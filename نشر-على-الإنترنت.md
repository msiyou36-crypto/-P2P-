# 🌍 نشر «سجل حوالات P2P» على الإنترنت

هذا الدليل ينشر نظامك على الإنترنت ليعمل **٢٤ ساعة حتى لو جهازك مغلق**، وتفتحه من أي مكان برابط + كلمة سرك.

- **الاستضافة:** Render (مجانية) — تشغّل الخادم.
- **التخزين:** Supabase (مجانية) — تحفظ بياناتك بأمان.

> ⚠️ **مهم:** بعد النشر تنتقل بياناتك ومفتاح Binance (للقراءة فقط) إلى السحابة، محميّة بكلمة سرك. مجلد `data` المحلي **لن يُرفع** (محمي بـ `.gitignore`).

---

## الخطوة 1 — رفع الكود إلى GitHub

الـ repo جاهز: `https://github.com/msiyou36-crypto/-P2P-`

جهّزتُ لك المشروع وعملتُ commit. يتبقّى **الرفع فقط**. افتح موجّه الأوامر داخل مجلد المشروع ونفّذ:

```bash
git push -u origin main
```

سيطلب منك تسجيل الدخول في GitHub (مرة واحدة). بعدها يظهر الكود في الـ repo.

---

## الخطوة 2 — تجهيز قاعدة Supabase

1. افتح مشروعك في Supabase → من القائمة اليسرى اختر **SQL Editor**.
2. اضغط **New query**، وانسخ محتوى ملف [`supabase-schema.sql`](supabase-schema.sql) كاملاً، ثم **Run**.
   - يُنشئ جدول تخزين واحدًا اسمه `kv` (محمي، لا يصله إلا خادمك).
3. الآن احصل على مفتاحين من **Project Settings ⚙ → API**:
   - **Project URL** — مثل `https://xxxx.supabase.co`
   - **service_role** secret — (تحت "Project API keys"، اضغط "Reveal"). **سرّي جدًا — لا تشاركه.**

> نستخدم `service_role` (وليس `anon`) لأن الخادم فقط يستخدمه، ولا يظهر أبدًا للمتصفح.

---

## الخطوة 3 — النشر على Render

### الطريقة الأسهل (Blueprint)
1. سجّل دخول في <https://render.com> (بحساب GitHub).
2. اضغط **New + → Blueprint**.
3. اختر مستودع `-P2P-`.
4. سيقرأ Render ملف `render.yaml` تلقائيًا. اضغط **Apply**.
5. سيطلب منك تعبئة المتغيرات الأربعة (انظر الخطوة 4).

### أو الطريقة اليدوية
1. **New + → Web Service** → اختر مستودع `-P2P-`.
2. اضبط:
   - **Runtime:** Node
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Plan:** Free
3. تابع الخطوة 4 لضبط المتغيرات.

---

## الخطوة 4 — ضبط المتغيرات (Environment Variables)

في Render، في قسم **Environment**، أضف هذه الأربعة:

| المتغير | القيمة |
|---|---|
| `SUPABASE_URL` | عنوان مشروع Supabase (Project URL) |
| `SUPABASE_KEY` | مفتاح **service_role** السري |
| `ADMIN_PASSWORD` | كلمة سر المسؤول (اخترها أنت) |
| `USER_PASSWORD` | كلمة سر المستخدم (اخترها أنت) |

ثم اضغط **Create / Deploy**. انتظر دقيقتين حتى تكتمل، وسيظهر رابط مثل:
```
https://sijil-p2p.onrender.com
```

---

## الخطوة 5 — أول استخدام

1. افتح الرابط → ستظهر شاشة الدخول مباشرة (كلمات السر ضُبطت من المتغيرات).
2. ادخل كـ **مسؤول** بكلمة سر `ADMIN_PASSWORD`.
3. من القائمة **☰ → ⚙ الإعدادات**: ألصق مفتاحَي Binance (API + Secret) بصلاحية «إتاحة القراءة» فقط، ثم احفظ.
4. من القائمة **☰ → ⟳ مزامنة**: ينزّل سجلّك من المنصة ويحفظه في Supabase.

تمّ! صار نظامك على الإنترنت، تفتحه من أي جهاز. 🎉

---

## أسئلة سريعة

**النظام بطيء أول فتح بعد فترة؟** الخطة المجانية في Render «تنيّم» الخادم بعد ١٥ دقيقة خمول، فيتأخّر أول طلب ~٣٠ ثانية ثم يشتغل عادي. لتفادي ذلك: خطة مدفوعة رمزية، أو خدمة تُبقيه صاحيًا.

**هل ما زال يعمل محليًا؟** نعم — بدون متغيرات Supabase يعمل النظام محليًا بالملفات كالمعتاد (شغّل `تشغيل.bat`).

**كيف أغيّر كلمات السر لاحقًا؟** من داخل النظام: **☰ → 🔑 تغيير كلمات السر** (للمسؤول).

**نسيت كلمة السر؟** غيّر قيمة `ADMIN_PASSWORD` في Render ثم أعد النشر — لكنها تُطبّق فقط إن لم تكن مضبوطة؛ لإعادة ضبطها امسح صفّ `config` من جدول `kv` في Supabase ثم أعد النشر.
